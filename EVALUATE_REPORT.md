**Finding 3: LiteLLM monkey-patch has no removal signal**
- `backend/traders/models.py:41` patches `AnthropicConfig._is_opus_4_6_model` with comment "Drop when LiteLLM 1.84+ ships" but no version check guards the patch. If LiteLLM removes the method, the patch silently adds a new attribute.
- Severity: Medium | Effort: Low | Impact: Medium

**Finding 5: No CI pipeline**
- No `.github/workflows/`, `Makefile`, or pre-commit hooks. 101 tests run in ~10s — fast enough for CI.
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 6: StopIteration as failure signal**
- `backend/arena/arena.py:79` uses `next(...)` which raises `StopIteration` for unknown reasoning labels. In Python 3.7+, this has special semantics in generators. Should use `next(..., None)` with explicit check.
- Evidence: `arena.py:79`, caught at `app.py:274`
- Severity: Medium | Effort: Low | Impact: Low

**Finding 15-19: Minor architecture items**

### Security & Compliance

**Finding 1: DEV_MODE grants blanket admin privileges — RESOLVED**
- `backend/auth.py`: `verify_admin` now requires a separate `DEV_ADMIN=true` env var for the admin bypass. `DEV_MODE` alone grants auth bypass (view access) but not admin privileges. Two env vars must leak to production for the footgun to fire. `check_auth_config()` logs differentiated warnings based on `DEV_ADMIN` state. Two new tests verify the gate.
- Status: **RESOLVED**

**Finding 10: No Content-Security-Policy header**
- `app.py:228-234` sets four security headers but omits CSP. Without CSP, XSS has no browser containment.
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 11: JWT tokens in localStorage**
- `apiClient.ts:21-22` and `auth.ts:76-78` store access and refresh tokens in `localStorage`. These are readable by any XSS. Combined with missing CSP, an XSS vector could exfiltrate both tokens.
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 28-32, 41-42: Minor security items**
- X-Request-ID unsanitized. Ticker input unvalidated. HSTS over HTTP. Auth check order in DELETE. No Permissions-Policy. In-memory lockout lost on restart. SSE tickets not shared across workers.
- Severity: Low | Effort: Low-Medium | Impact: Low

### Performance & Scalability

**Finding 12: Per-tick N+1 query pattern**
- `arena.py:249-278` executes ~28 SQLite SELECTs per tick (4 traders x ~6 queries each + 4 from `_all_held_tickers()`). `portfolio_value()` internally re-fetches `holdings()` and `cash()` already queried by `_snapshot()`, adding 8 redundant queries.
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 13: Single-consumer SSE queue**
- `arena.py:339-345` uses one `asyncio.Queue`. Two browser tabs race on `events.get()` — each event goes to one consumer, not both.
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 14: `get_state_impl()` generates ~14 queries per call**
- Each `get_state` call fetches own holdings, cash, portfolio_value (2 more queries), plus 3 rival holdings + 3 rival portfolio_values (6 more). Total: ~14 SELECTs per invocation from concurrent async trader tasks.
- Evidence: `tools.py:52-53,67`
- Severity: Medium | Effort: Medium | Impact: Medium

**Finding 33-39: Minor performance items**
- `execute_trade` blocks event loop (sub-ms). `get_config` disk read. `load_users` repeated calls (mtime mitigated). Heatmap forced reflow. Chart `getComputedStyle`. Portfolio redundant holdings. Price cache benign race.
- Severity: Low-Medium | Effort: Low-Medium | Impact: Low
